## CLI Validation

### doctor
- [ ] `cloudosctl doctor` succeeds on a clean development machine
- [ ] Each failure includes: what failed, why it matters, how to fix
- [ ] Doctor is read-only (no side effects)

### deploy
- [ ] `cloudosctl deploy <git-url>` creates an application
- [ ] Deployment polling shows progress
- [ ] Successful deployment prints "Open in browser? [y/N]"
- [ ] Failed deployment shows error message
- [ ] Works with all 7 example applications

### logs
- [ ] `cloudosctl logs <app>` shows recent logs
- [ ] `cloudosctl logs <app> -f` streams logs in real-time
- [ ] `cloudosctl logs <app> -n 100` shows last 100 lines
- [ ] `cloudosctl logs <app> -d` downloads logs to a file

### ps
- [ ] `cloudosctl ps` lists all applications
- [ ] Output includes ID, Phase, Health, URL columns

### open
- [ ] `cloudosctl open <app>` resolves URL
- [ ] Opens browser on macOS, Linux, Windows
- [ ] Prints URL if browser launch fails

### status
- [ ] `cloudosctl status <app>` shows all 7 sections
- [ ] `cloudosctl status <app> --json` outputs valid JSON
- [ ] `cloudosctl status <app> --watch` refreshes every 2 seconds

### timeline
- [ ] `cloudosctl timeline <app>` shows latest deployment steps
- [ ] `cloudosctl timeline <app> -n 3` shows deployment #3
- [ ] Step status icons are correct (✓, ✗, ◌, →, ⊘)

### compare
- [ ] `cloudosctl compare <app> 1 2` shows side-by-side diff
- [ ] Changes section shows only what changed
- [ ] Unchanged section shows what stayed the same

### version
- [ ] `cloudosctl version` returns valid version string

## API Validation

- [ ] `GET /api/v1/health` returns 200 with component health
- [ ] `GET /api/v1/version` returns version metadata
- [ ] `GET /api/v1/resources/Application` returns application list
- [ ] `POST /api/v1/resources/Application` creates an application
- [ ] `GET /api/v1/applications/{id}/logs` returns log events
- [ ] `GET /api/v1/applications/{id}/logs/stream` returns SSE stream
- [ ] `GET /api/v1/applications/{id}/logs/download` returns text file
- [ ] `GET /api/v1/applications/{id}/deployments/{n}/timeline` returns steps
- [ ] `GET /api/v1/applications/{id}/deployments/compare` returns comparison
