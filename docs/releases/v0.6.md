# CloudOS v0.6 Release Notes

> **Release Candidate** — July 2026

## Highlights

CloudOS v0.6 is the first feature-complete version of CloudOS. From a single
command to a running, observable application — with full deployment visibility,
no cloud account required.

### From Zero to Running in ~3 Minutes

```bash
curl -fsSL https://cloudos.io/install.sh | sh
cloudosctl doctor
cloudosctl deploy https://github.com/cloudos-examples/go-api
cloudosctl open go-api
```

### Full Deployment Observability

Every deployment is transparent. See exactly what happened, how long each step
took, and where it failed:

- **Live Logs** — stream build, deploy, and runtime logs in real-time
- **Deployment Timeline** — step-by-step view of every deployment
- **Deployment Comparison** — side-by-side diff of any two deployments
- **Deployment Report** — 19-field structured summary per deployment

### 7 Certified Stacks

| Stack | Status |
| :---- | :----- |
| Go | Full certification (build, run, health, logs, metrics) |
| Node.js | Detection verified |
| React (Vite) | Detection verified |
| Next.js | Detection verified |
| Python (Flask) | Detection verified |
| Laravel / PHP | Detection verified |
| Static HTML/CSS | Detection verified |

### Two Runtimes

- **LocalRuntime** — runs applications as local processes (default, no Docker needed)
- **OCI Runtime** — runs applications in Docker containers (better isolation)

The OCI Runtime proved that the Runtime contract is stable and replaceable —
a second implementation plugged in with zero changes to the core pipeline.

### Application-Centric Dashboard

The web dashboard (React, dark-first) provides:
- Application grid with status, health, and quick actions
- Detail views with 5 tabs: Overview, Deployments, Timeline, Logs, Settings
- Real-time log streaming with pause, search, and download

## New Features

### CLI

| Feature | Description |
| :------ | :---------- |
| `cloudosctl doctor` | 14 read-only environment checks with remediation |
| `cloudosctl deploy` | Deploy from Git URL with deployment polling |
| `cloudosctl logs` | Snapshot, stream (SSE), and download logs |
| `cloudosctl ps` | List all running applications |
| `cloudosctl open` | Open application in default browser |
| `cloudosctl status` | Terminal dashboard with `--json` and `--watch` |
| `cloudosctl timeline` | Step-by-step deployment timeline |
| `cloudosctl compare` | Side-by-side deployment comparison |
| `cloudosctl version` | Version information |

### Dashboard

- Application list as default landing page
- Application detail with 5-tab navigation
- Real-time log viewer (SSE streaming, pause/resume, search, download)
- Visual deployment timeline (framer-motion staggered animation)
- Deployment comparison view
- Application settings (read-only metadata)

### API

- `GET /api/v1/resources/Application` — list all applications
- `GET /api/v1/resources/Application/{id}` — get application details
- `POST /api/v1/resources/Application` — create (deploy) application
- `GET /api/v1/applications/{id}/logs` — log snapshot
- `GET /api/v1/applications/{id}/logs/stream` — SSE log stream
- `GET /api/v1/applications/{id}/logs/download` — log download
- `GET /api/v1/applications/{id}/deployments/{n}/timeline` — deployment timeline
- `GET /api/v1/applications/{id}/deployments/compare` — deployment comparison

### Platform

- Frozen contracts at v1.0: Runtime, Buildpack, Workflow APIs
- OCI Runtime (Docker) — second runtime implementation, proving contract substitutability
- Deployment Report — 19-field structured metadata per deployment
- Certification framework — 7 stack certification tests + harness

## CLI Improvements

- Every failure message includes remediation: what failed, why it matters, how to fix
- Doctor is read-only — never modifies the system
- `deploy` command polls for completion and offers to open browser
- `status -–watch` provides live-updating terminal dashboard
- `logs -d` downloads logs to a file for offline analysis
- All commands produce consistent error output with API error codes

## Known Limitations

### v0.6 Release Candidate

| Limitation | Impact | Workaround |
| :--------- | :----- | :--------- |
| No authentication | Anyone with API access can deploy/modify apps | Run on localhost only |
| No persistent storage | Stopped applications lose state | Keep apps running |
| Single-node only | Cannot scale across machines | Use for local dev |
| No hot-reload | Code changes require re-deploy | Run `cloudosctl deploy` again |
| Windows requires WSL2 | No native Windows support | Use WSL2 with Ubuntu |
| Plugin SDK not available | Cannot build custom buildpacks/runtimes yet | Use built-in stacks |
| Static sites don't produce logs | No application log output | Use the dashboard health check |
| No HTTPS | All endpoints are HTTP | Development only |

## Upgrade Notes

### From v0.5 to v0.6

**Breaking changes:**
- The kernel (`go run ./tools/cloudos`) now requires the Control Plane API
  to be running before `cloudosctl` commands will work
- Application resources now include a `DeploymentReport` in their status

**New dependencies:**
- `cloudosctl doctor` checks for Git, Docker, and toolchains
- Dashboard requires a modern browser (Chrome 120+, Firefox 120+, Safari 17+)

**Migration steps:**
1. Pull the latest code: `git pull`
2. Rebuild: `go build ./...`
3. Run doctor: `cloudosctl doctor`
4. Restart the kernel: `go run ./tools/cloudos`
