# Architecture Decision Records (ADR)

This directory contains Architecture Decision Records for CloudOS.

Each ADR captures a significant architectural decision: the context, the
decision, alternatives considered, and the consequences.

## How to use

1. Create a new file with the next sequential number: `NNNN-title.md`
2. Follow the template below
3. Get team agreement before merging

## Template

```markdown
# ADR-NNNN: Title

## Context

What problem are we solving? What constraints or forces are at play?

## Decision

What architecture are we adopting? Be specific.

## Alternatives Considered

What other approaches did we evaluate? Why were they rejected?

## Consequences

What tradeoffs come with this decision? What does this enable or constrain?
```
