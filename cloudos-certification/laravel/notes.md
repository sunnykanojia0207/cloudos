## Architecture Notes

- Laravel buildpack detects by `composer.json` presence.
- Install command chains: `composer install && (copy .env.example .env) && php artisan key:generate`
- Start command: `php artisan serve --host=0.0.0.0 --port={port}`
- `APP_ENV` set to `production`, `APP_DEBUG` set to `false`.
- The `.env` file is created by copying `.env.example` if it doesn't exist.
- The `APP_KEY` is generated by `php artisan key:generate`.
- Output directory set to `public` (Laravel serves from the `public/` directory).

## Potential Issues

- PHP and Composer must be installed on the host system (not bundled with CloudOS).
- Some Laravel packages require Node.js/npm for frontend assets.
- Database migrations (`php artisan migrate`) are NOT automatically run — the user needs a pre-configured database.
- `php artisan serve` is development-only — production uses Nginx/FrankenPHP.
